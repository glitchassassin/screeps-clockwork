# WASM Boundary Efficiency Hypotheses

Research date: 2026-05-18

Node WASM benchmarks run with the Screeps V8 flags:

```sh
--no-liftoff --no-wasm-lazy-compilation --wasm-wrapper-tiering-budget=0
```

Scope: ideas should preserve the current TypeScript-facing API unless explicitly marked as a parking-lot item. In particular, `costMatrixCallback(room: string) => ClockworkCostMatrix | undefined` should remain valid.

## Sources

- [`wasm-bindgen` supported types](https://wasm-bindgen.github.io/wasm-bindgen/reference/types.html): overview of Rust types and JavaScript boundary representations.
- [`wasm-bindgen` number slices](https://rustwasm.github.io/docs/wasm-bindgen/reference/types/number-slices.html): numeric slices cross as typed arrays/views depending on direction and ownership.
- [`wasm-bindgen` receiving JavaScript closures](https://wasm-bindgen.github.io/wasm-bindgen/reference/receiving-js-closures-in-rust.html): documents the `js_sys::Function`/`call#` callback path currently used by search wrappers.
- [`js-sys` typed `Function`](https://wasm-bindgen.github.io/wasm-bindgen/reference/types/js-sys.html): typed JavaScript function wrappers and primitive coercion support.
- [`wasm-bindgen` exported Rust structs](https://rustwasm.github.io/docs/wasm-bindgen/contributing/design/exporting-rust-struct.html): exported structs become JS classes that hold wasm-side pointers and expose shims.
- [`wasm-bindgen` arbitrary data with Serde](https://wasm-bindgen.github.io/wasm-bindgen/reference/arbitrary-data-with-serde.html): high-level object materialization can lose to simpler representations, including JSON in some cases.
- [`wasm-bindgen` reference types](https://rustwasm.github.io/docs/wasm-bindgen/reference/reference-types.html) and [MDN `externref`](https://developer.mozilla.org/en-US/docs/WebAssembly/Reference/Types/externref): reference types can pass JS values without the older wasm-bindgen heap indirection, but runtime compatibility matters.
- [V8 WebAssembly compilation pipeline](https://v8.dev/docs/wasm-compilation-pipeline) and [V8 speculative Wasm optimizations](https://v8.dev/blog/wasm-speculative-optimizations): benchmark results can depend on tiering, flags, and current cross-language inlining limitations.

| H1  | Replace `get_cost_matrix.call1(null, room)` with an inline JS helper that directly invokes `callback(room)`.                                                         | None for TS callers. Compare repeated `bfsMultiroomDistanceMap`/`astarMultiroomDistanceMap` calls with and without cache. | Search callbacks happen once per discovered room, so this helps repeated calls more than one large search. Cache is bounded naturally by room-name space used by the process. | Accepted         |

Include invalid-return tests and callback-heavy synthetic cases.                                                                | Relies on wasm-bindgen-generated pointer details. Must not transfer ownership. Still clone the matrix unless a separate ownership strategy is proven safe.                    | Rejected         |

| H7  | Reduce short typed-array allocations in TS wrappers for start and destination inputs.                                                                                | None.                                                                              | Wrappers currently allocate intermediate JS arrays with `map`/`reduce`, then create `Uint32Array`s for wasm-bindgen numeric slices. ## Test Results: 2026-05-18

| ID  | Result                                                                                                                                                         | Decision                                                                                    |
| --- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| H1  | Screeps-flag rerun regressed all main search cases: Dijkstra/A\* were about 0.62x-0.88x of baseline and BFS about 0.87x-0.95x.                                 | Rejected; direct callback invocation is worse than `Function.call1` under the target flags. |
| H2  | Screeps-flag rerun was mixed, not better: BFS rose about 1-2%, but A\* empty multiroom was 0.96x and private-sector Dijkstra/A\* were 0.98x.                   | Rejected and reverted.                                                                      |
| H3  | Screeps-flag wrapper bench: cached room names were 2.50x faster for 81 callback-name conversions.                                                              | Accepted. Implemented `fromPackedRoomNameCached` in TS wrappers.                            |
| H4  | Screeps-flag rerun was mixed: pointer-returning callback was mostly flat, but A\* realistic cases regressed to about 0.96x-0.97x.                              | Rejected and reverted.                                                                      |
| H5  | Screeps-flag rerun remained mixed, with private-sector A\* at 0.86x and private-sector BFS at 0.93x.                                                           | Rejected and reverted.                                                                      |
| H6  | Screeps-flag reference-types build helped some empty multiroom/BFS cases, but A\* empty room and realistic W1N1 regressed and path/flow-field results varied.  | Rejected for current build scripts.                                                         |
| H7  | Screeps-flag wrapper bench: manual typed-array fill was 1.63x faster for eight starts, 1.58x faster for one destination, and 1.75x faster for 64 destinations. | Accepted. Implemented manual `packPositions` and `packDestinations` helpers.                |
| H8  | Prototype bulk `ClockworkCostMatrix.fromArray(Uint8Array)` was 5.25x faster than 2,500 `set` calls.                                                            | Validated, but not kept because it adds API surface outside this existing-API pass.         |
| H9  | No current richer result-shape change exists to benchmark.                                                                                                     | Keep as design guardrail: benchmark packed/Rust-backed outputs before future object APIs.   |
