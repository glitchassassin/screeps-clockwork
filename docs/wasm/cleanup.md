# WASM Object Cleanup

Clockwork relies on the `FinalizationRegistry` support generated by `wasm-bindgen` to release Rust-backed objects once their JavaScript wrappers are no longer reachable.

This means Clockwork objects should be treated like ordinary JavaScript objects:

```ts
const distanceMap = bfsMultiroomDistanceMap(start, {
  costMatrixCallback: room => getTerrainCostMatrix(room),
  maxRooms: 2
}).distanceMap;
```

There is no Clockwork-level `ephemeral()` helper and normal code should not manually call `.free()`. Keep references for as long as the data is useful, and remove objects from your own arrays, maps, and caches when they are no longer needed so the JavaScript runtime can collect them.

Garbage collection is nondeterministic. If you cache many large objects, bound the cache or clear stale entries explicitly; the cleanup happens after the runtime determines those wrappers are unreachable.
